[PS2] Graphics Synthesizer

 

Statistics:

  • 2.4 giga-pixel
  • fill rate statistic:
  • 150 million points
  • 50 million sprites
  • 75 million untextured triangles
  • 37.5 million textured triangle

Supported Rendering:

alpha blending, z Testing, bi- and tri-linear filtering

Embedded DRAM

Features a 2048-bit-wide interface to prevent bottleneck when doing pixel filling. This greatly reduces bandwidth demands on the DRDRAM and also makes graphic synthesizer the one unit that has the highest bandwidth portion of the memory system in the PS2. The unit also includes separate memories dedicated to individual functions to inexpensively achieve greater memory bandwidth for the entire system.

  • System bandwidth: 48GB/sec
  • Frame buffer bandwidth: 38.4GB/sec
  • Texture bandwidth 9.6GB/sec

​Pixel Processor

Runs at 150mhz, with 4MB embedded DRAM and a fill rate of 1.2GB/sec for textured, and 2.4pixel/sec for untextured. The entire unit is only 279 mm2 making it very light weight while still including 16.8 million transistors. It includes a GS coordinate system frame buffer as use as a 16-bit coordinate system. This means that 12-bit integer, and 4-bit fractions, with the full range being 0 to 4095.9375. The center is mostly specified as (2048, 2048). The scissoring area is specified based relative to (2048, 2048). GS coordinate scissoring is when there is a vertex X and Y that are both 16bit thus it won’t work outside the range. There is no hardware clipping, but there is vector unit clip instruction.

 

Reading Questions

  1. What are the dimensions of the graph?
  2. Is there any hardwear clipping in this type of coordinate system?
  3. What is the bandwidth of the embedded DDRM?
