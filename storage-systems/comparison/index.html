Comparison

Price

Source: “SSD prices in steady, substantial decline.” The Tech Report.

Prices of SSD’s have dropped from around $2/GB in 2011 to $1/GB or less in 2013 - a 50% decrease in only a couple years. Although the cost of hard drive storage remains much less, around $0.1/GB (Source: http://ns1758.ca/winch/winchest.html), the affordability of modern SSD’s make them an attractive option for any PC user and can noticeably increase performance for relatively little cost. As an example of current storage prices, a Samsung 512GB SSD costs $520 while a Western Digital 3TB HDD costs $135. That results in $1.02/GB for solid state storage and $0.04/GB for hard disk storage, or in other words, solid state storage is approximately twenty-five times as expensive as hard disk storage in its current state. However, as their prices continue to go down, it is likely that SSD’s will replace hard drives entirely in the future which makes them an important component of architecture to any computer user.

 

Technology

There are two main types of flash memory; NOR and NAND.  NOR flash memory allows random access but suffers from longer erase/write timies.  NAND flash memory requires sequential access but has faster erase/write times, a higher memory density, lower cost, and higher endurance.  Virtually all SSDs are made of NAND flash memory.

There are two main standards for NAND flash memory: ONFI and toggle-mode DDR NAND.  ONFI is backed by Intel, Micron Technology, and Sandisk.  Toggle-mode DDR NAND is backed by Samsung and Toshiba.  ONFI has gone through several revisions: ONFI 1.0 through 3.1.  1.0 supports speeds up to 50 MB/s, 2.0 with 133 MB/s, and 2.1 with 166 MB/s or 200 MB/s, and 3.0 with speeds up to 400MB/s.  The most recent Samsung toggle-Mode DDR NAND interface (PM841 Series) has read speeds up to 520 MB/s and writes up to 330MB/s.

NAND flash memory is also split into two more types: multi-level cell (MLC) or single-level cell (SLC).  MLC SSDs are more prone to errors, but have higher data density.  MLC technology is widely used in consumer SSDs. These SSDs typically use 25mm cells which exist in four states, allowing two bits of information.  SLC SSDs are used for data stability in enterprise-level applications.  They utilize 50mm cells with one bit of information stored with two possible states.

 One disadvantage of SSDs is that, unlike HDDs, memory in solid state drives must be erased before they can be written to again. A way to compensate for this is a method of garbage collection that utilizes TRIM commands. Instead of erasing sections of memory right before it needs to be written to, unused storage space is actively marked for deletion which gets cleaned up by TRIM.

Source: “How to Enable TRIM Support for Your SSD” Corsair Blog.

 

Solid state drives operate using a controller. A controller is an embedded processor inside the SSD which bridges the flash memory to the input/output interface of the system. This processor executes firmware that is created by the manufacturer to operate the onboard flash memory as well as run other diagnostics and garbage collection which can have an impact the overall performance of an SSD. One of the benefits of having an SSD is a feature called SMART (Self-Monitoring, Analysis and Reporting Technology) which monitors the health of the drive and can anticipate a failure. This feature is only possible on solid state drives because of the controller so it is not available on hard disk drives.

Source: “SSD Controller.” Storage Review. http://www.storagereview.com/ssd_controller

 

Performance

The inherent advantage of solid state drives is that they have no moving or mechanical parts that would limit the speed of reading/writing data. The easiest way to see this difference is through comparing performance benchmarks of both storage systems. It is most apparent in the random writes and reads. While a traditional HDD would have to move the disk head relatively large distances over the platter, an SSD would have to just transfer the block containing the randomly-chosen data.

Source: AnandTech Bench Western Digital Red WD30EFRX 3TB

The only case where a HDD outperforms a SSD is when storage is preferred over performance (additionally, data on HDDs can last longer when sitting unpowered, data on SSDs only last around a year unpowered).  In all other cases, a SSD should be chosen. For storage, a 7200RPM HDD has around the same IOPS as a high-performance HDD, but vastly lower cost per GB. High-RPM HDDs are being squeezed out of the market by much cheaper performing ($/IOPS) SSDs and much cheaper storage ($/GB) HDDs.

Math for the curious (adapted from example on page 395 in the 4th edition) (assume 64kb block size):

    • WD VelociRaptor: 7.1ms + 0.5/10000RPM + 64KB / (200MB/s) = 10ms avg read time

1/10ms = 100 IOPS

$179.99/100 IOPS  = $1.79/IOPS

    • WD RED = 8.88ms + 0.5/5400 RPM + 64KB/(145 MB/s) = 14.8ms avg read time

1/14.8ms = 67.05 IOPS

$134.99/67.05 = $2.01/IOPS
