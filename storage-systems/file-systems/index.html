File Systems

If you create something on a computer or find something interesting, it is likely that you will want to save it and find it again later. It could be saved to the cloud, but to access your data without an Internet connection, a block device will be necessary. This means you're going to need a file system.

Yes, file systems. This mystical concept your operating system encounters ensures your data can be found again. Simply, the file system organizes the data you need to store. But this is much more complicated than one might first expect and begins with the physical storage device. For the focus of this page, we will assume the file system organizes data to be stored on a rotating magnetic platter hard disk drive (HDD). 

Why?

A file system is needed for effectively using a block device, like a hard drive, for data storage. Block devices read and write data in arbitrary chunks at a time. For typical hard drives, the physical block size is 512 bytes. For hard drives, physical blocks are called sectors and the size of each sector is determined by the design of the device. The block size represents the smallest chunk of data that can be read or written from the device. For simple file systems, this means that every file will take at least 512 bytes to store on the disk and large files will use multiple sectors. 

However, with modern file systems, block sizes are often 4 - 8 kilobytes, larger than the physical sector size, and can be even larger. Also, modern file systems have additional features, which allow multiple files to be stored within one block for more efficient utilization of disk space. 

I Just Want My Stuff Back

It's ok. Following the rules of the file system, your data was thrown into one or more blocks on the drive. Now, we just need to be able to find it again. This is why we have file systems. Early file systems were not designed with the same performance considerations we have today. These file systems were storing data on floppy disks smaller than a megabyte or on hard drives with total capacity measured in megabytes. One of these file systems was FAT or File Allocation Table; this is still used today on USB flash drives because it is simple and lightweight. The way it works is described by its name: the file system takes up space on the drive to store a table identifying where files have been placed. This is a simple look-up table mapping blocks on the drive to files. To find a file, go check the table and get a location on the disk to find the contents of the file.

Also, the file system basically put the file contents in the first empty block. If the file was larger than one block, it may now be spread across different areas of the drive, each block in a different spot, but some may be contiguous. This means the file is now fragmented. Checking different spots to retrieve the entire file takes more time. This design was fine, but then disks increased in capacity to hundreds of gigabytes and the number of files stored on the drive increased substantially. Modern file systems required a different approach to improve the speed of retrieving files.

What Else Does It Do?

Aside from organizing files on the hardware level, it helps the user organize files. The file system defines how folders work so users can visually sort files into different categories, without moving the files to different physical locations on the disk. Also, it facilitates storage of file metadata (data about a file). Metadata contains all sorts of useful information: access times, creation date, permissions, file type, attributes, and more.
