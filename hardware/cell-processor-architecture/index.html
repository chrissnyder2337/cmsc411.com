Cell Processor Architecture

 

Specifications

The Cell is comprised of the following elements:

  • 1 Power Processor Element (PPE).
  • 8 Synergistic Processor Elements (SPEs).
  • Element Interconnect Bus (EIB).
  • Direct Memory Access Controller (DMAC).
  • 2 Rambus XDR memory controllers.
  • Rambus FlexIO (Input / Output) interface.

The Power Processor Element (PPE)

The PPE is the primary core that directs tasks to the SPEs. The PPE is in charge of running the operating system and the majority of the applications, but it redirects computationally intensive work to the SPEs. The PPE is a 64 bit dual issue, dual threaded processor. It uses the RISC-based PowerPC instruction set, which only supports in-order instructions. This differs from most modern processors, which usually use out-of-order execution to maximize the number of instructions being executed at any time. The PPE also has a 512 KB locking cache, meaning it can 'lock' areas of the cache to prevent them from being overwritten. 

 

The Synergistic Processor Elements (SPEs)

A SPE consists of a single vector processor that acts as an independent processor. There are 8 SPEs on one Cell, each containing 128 registers. There are also 4 single-precision floating point units, and 4 integer units, each capable of 32 GOPS at 4GHz. Each SPE has comparable performance to a high-end single-core desktop CPU, and consumes less that 5 watts at 4GHz. 

The SPE features a local store in favor of a cache. A cache takes time to update and has size limitations, which reduce the overall processing speed and effeciency. So instead, each SPE has a single 256 KB register for local storage space. These special registers have a read/write speed of up to 64 GB/sec. Overall, this reduces the physical complexity and increases performance, but as you can imagine, keeping all data in one large register makes programming difficult. 

SPEs, like PPEs, are in-order processors. They have no Out Of Order (OOO) instruction capabilities. Therefore, optimizing code is left entirely to the compiler. However, since the SPEs have 128 registers, loop unrolling and other optimization techinques are easier than on a traditional CPU. 

Each SPE is capable of 4 32-bit operations per clock cycle. To take advantage of this parallelism, programs must be 'vectorized' so that multiple instructions can be exectued at once. This is similar to pipelining or multi-threading. Vectorizing is usually done manually, although some compilers offer an 'autovectorizing' feature. 

Stream Processing
An interesting feature of Cell is that the SPE's can be chained together in a sort of 'stream'. This way, one or more SPEs can be devoted to a single job, and then the output is fed directly to the next SPE from the local store. This allows each SPE to be a 'special-purpose' element in a longer chain. 
 
Although typically not needed, Cell features high-bandwidth between SPEs and an interconnect system that allows multiple streams at a time. This allows transfer speeds in the 100s of GB/sec. If more computing power is needed, multiple Cells can be combined to form even larger 'sterams'. 

This type of performance and parallelism is very similar to modern GPUs.

 

Inside the Cell

Although the Cell is optimized to perform and excel exceptionally well for highly-computable and intensive tasks, it is also designed as a general purpose processor. While the PPE is a comparable to a traditional processor, the SPEs are optimized to accelerate specific parts of code over others.

Over the last 20 years, traditional CPU architecture design emphasized increasing, along with the clock speed, the Instructions Per Cycle. This has been implemented through installing additional transistors which result in many more instructions to be complicatedly executed Out Of Order (OOO).

However, the PPE takes on an outside-the-box approach, by not incorporating OOO hardware. This is because OOO CPUs utilize a high number of transistors to achieve their goals, which requires immense power and cooling capabilities. Cooling has become increasingly hard due to the multitude of transistors, which leak electrons when not in use, resulting in additional power consumption, and ultimately dissipating more heat. This physical limitation of boosting clock speed for performance is the motivation behind multi-core processors. The PPE Cell incorporates a RISC architecture within its dual-threaded, high clocked, low power core. It deliberately consists of a simple design to get around the heat issues seen previously with OOO architectures.

 

Branch Predictions

The PPE includes branch predictors, as in other popular architectures, but here they are not emphasized as much. Branch prediction is analogous to OOO hardware, in the sense that it can only boost performance sometimes. The performance of the branch predictor is dependent on the type of code executing, which poses an additional challenge of programming for the PS3.
