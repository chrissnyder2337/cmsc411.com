In a block level storage device, raw storage volumes are created, and then the server-based operating system connects to these volumes and uses them as individual hard drives. This makes block virtualization extrememly flexible and versatile, being able to be used for almost any kind of application. This includes file storage, database storage, and virtual machine file system volumes. </p> <p><strong><u>The Benefits</u></strong></p> <p>As previously mentioned, block virtualization's biggest benefit is its versatility and flexibility. For example, block virtualization systems allow files to be shared without using the native operating system, as it essentially creates a blank hard drive which you can do anything with. It is also much simpler to set up backup systems than with other virtualization techniques. Since each volume acts as a normal hard drive, no special back up steps need to be taken. These benefits make it used in a variety of applications, such as databases, VMware, and, using specific  server configurations, even server booting.</p> <p><strong><u>The Drawbacks</u></strong></p> <p>The downside to block virtualization lies in its complexity in management. Administrators are given many more roles, and need to take a much more active role in discerning the health and maintenance needs of the storage system. Systems also need to be managed on a per server basis, rather than as a network of servers as a whole. These requirements put large strains on systems management teams, often leading to higher human costs, as well higher maintenance costs.</p> </div></div></div> Sat, 04 May 2013 04:21:28 +0000 ipersons Specifically, these features are seen in ext4, XFS, and btrfs. HFS+ for the Macintosh OS is similar. Some of these features exist in NTFS under Windows, but others are unique.</p> <h3>Inodes</h3> <p>The name means <em>index nodes</em>. A property of <a href="/node/141#*nix">*nix</a> file systems. They are data structures which contain all the information about a file other than the filename and file content. They contain file metadata, blocks/extents associated with a file, and the number of <a href="/node/141#links">links</a> associated with a file.</p> <h3>Journaling</h3> <p>Basically a log consisting of changes made to the file system. This reduces the risk of data loss after unexpected errors occur, such as, power failures or operating system faults. Without journaling, these events are more likely to cause data loss and data corruption, and requires that the file system undergo a check (using the utility fsck on <a href="/node/141#*nix">*nix</a>) to find and correct errors from partly written data and inconsistencies in the file system structure.</p> <h3>Extents</h3> <p>Extents are a set blocks which are <a href="/node/141#contiguous">contiguous</a> at the file level and at the physical level. Inodes can point to extents instead of blocks when storing the locations of file data. Extents reference a range of blocks which contain the data for the file, so this leads to faster <a href="/node/141#IO">I/O</a> operations and reduces the number of block pointers necessary to reference the entire file. Since fewer references are needed to store the file, less disk space is needed to store the inodes. Additionally, this reduces complexity resulting in faster creation and deletion of very large files. Using extents encourages the use of <a href="/node/141#contiguous">contiguous</a> blocks when allocating space for files.</p> <h3>B-Tree Structure</h3> <p>B-Trees are a tree-based data structure, of a general form similar to binary trees. This data structure is a much faster alternative to the lookup table older file systems (like FAT) used, when indexing large drives and many files. XFS uses a variation, B+ trees to keep track of free space on the drive. In XFS two B+ trees are used, one tracks the length of free extents, while the other tracks the starting block of free extents. This design makes XFS very fast at creating large files because it can efficiently locate adequate free space for the file.</p> <h3>64-Bit Addressing</h3> <p>Allows up to 16 extabytes of storage to be accessible.</p> <h3>Online Defragmentation</h3> <p>Defragmentation involves shifting data to different areas on the drive to minimize the number and amount of file <a href="/node/141#fragmented">fragmentation</a>. Online defragmentation allows this process to occur while the file system is in use. The need for this is somewhat reduced by the use of extents in preventing fragmentation from occurring until the drive begins to fill.</p> </div></div></div> Sat, 04 May 2013 02:52:49 +0000 dsemick
