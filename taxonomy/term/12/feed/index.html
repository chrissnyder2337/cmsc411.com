A computer on which a hypervisor is running is called a <em>Host Machine</em> and the several system virtual machines (OSs) managed by the hypervisor are called <em>Guest Machines.</em> A hypervisor allows each guest OS to execute as though it owns the host's processor, memory, and other hardware resources (when in reality those resources are shared among all guest OSs).</p> <p>            A guest OS must behave on a VM exactly as if it were running on native hardware except for performance-related behavior or limitations of fixed resources shared by multiple VMs. Also, guest OSs should not be able to change allocation of real system resources directly.[7]</p> <p>            There are 2 types of hypervisors:[8]</p> <ul><li>Type 1 (<em>native </em>or <em>bare metal</em>) hypervisors run directly on the system hardware. Examples include Xen and POWER5™.</li> <li>Type 2 (<em>hosted</em>) hypervisors run on a host operating system that provides virtualization services, such as I/O device support and memory management. Examples include VMware GSX Server and Microsoft<sup>®</sup> Virtual Server.</li> </ul><p>Type 1 hypervisors are typically the preferred approach because they can achieve higher virtualization efficiency by dealing directly with the hardware. Type 1 hypervisors provide higher performance efficiency, availability, and security than type 2 hypervisors. Type 2 hypervisors are used mainly on client systems where efficiency is less critical. Type 2 hypervisors are also used mainly on systems where support for a broad range of I/O devices is important and can be provided by the host operating system.</p> <p><img alt="" src="/sites/default/files/files/vm/hypervisors.png" style="width: 600px; height: 342px;" /></p> </div></div></div> Sat, 17 Aug 2013 00:09:00 +0000 mikasent 271 at http://cmsc411.com http://cmsc411.com/software/hypervisors#comments Virtual Machines: An Overview http://cmsc411.com/software/virtual-machines-overview <div class="field field-name-body field-type-text-with-summary field-label-hidden"><div class="field-items"><div class="field-item even" property="content:encoded"><p>Advances in technology have given rise to new ideas for testing and implementing new technologies.  Arguably one of the most important methods discovered to achieve this is virtualization.  According to IBM, the pioneers of this technique, <em>virtualization</em> is a “technique for hiding the physical characteristics of computing resources from the way in which other systems, applications or end users interact with those resources” [1]. The concept of virtualization can be applied to applications like the java virtual machine (JVM), operating systems, servers, and even networks. This page outlines a few applications of virtualization and provides an overview of virtual machines.</p> <p><u><strong>History</strong></u></p> <p>In 1963 the Massachusetts Institute of Technology (MIT) received a $2 million grant for their Mathematics and Computation project (later renamed Multiple Access Computer) which aimed to further the world of computer science in the areas of operating systems, artificial intelligence, and computational theory.  In order to conduct their research, MIT needed computer hardware capable of supporting multiple simultaneous users.  After presenting their request to both IBM and GE they decided GE was most willing to give them the machine they wanted.  The shock of losing the contract and the news of Bell Lab’s need for a similar system sparked IBM’s first multiple simultaneous user machine, the CP-40.  After the CP-40 came the CP-67; the first main frame to support virtualization.  Running on the CP-67 was an operating system called the Control Program/Control Monitor System (CP/CMS) which was designed to be user interactive and capable of creating virtual machines.  After four years of perfecting the machine it was finally released to the public in 1972 and thus the time of virtual machines began.  Since then, virtual machines have proved themselves to be very useful in the study and enhancement of the field of computer science. [5]</p> <p><u><strong>What is a VM?</strong></u></p> <p>A <em>virtual machine</em> <em>(VM)</em> is a “software emulation of a physical computing environment” [1].  This environment hides the original computing platform from any program operating within it and operates as if it was running directly on the hardware.  Therefore it has access to all the physical machines resources like network access, display, keyboard, and main-memory.</p> <p><strong><u>Advantages [4]</u></strong></p> <ul><li>Multiple OSs able to run on a single physical machine</li> <li>Older applications can be run on newer machines</li> <li>Quick disaster recovery because of the ease of taking snapshots</li> <li>Virtual machines are easily transferred from one physical machine to another</li> <li>Decreases the number of physical machines purchased</li> <li>Less physical machines means less power consumption</li> </ul><p><u><strong>Disadvantages [4]</strong></u></p> <ul><li>Precise specification of current and future requirements</li> <li>Multiple virtual machines running on a single physical system require more hardware resources</li> <li>Necessary hardware must be present in order for optimal performance</li> <li>All virtual machines will be lost in the event of a host OS or host hardware failure</li> </ul><p><u><strong>Desktop Virtualization</strong></u></p> <p>One very common corporate use of virtualization is <em>desktop virtualization</em>.  Desktop virtualization involves moving the operating system of a desktop computer to a virtual computer running on a server.  Users are still able to customize their desktop environment and also access their virtual computer from different locations.  Organizations with many users such as corporations or universities often utilize this function to provide a greater range of access to their users’ information</p> <p><img alt="" src="/sites/default/files/files/vm/virtualdesktop.jpg" style="width: 500px; height: 258px;" /></p> <p><u><strong>Server Virtualization</strong></u></p> <p>Servers can also utilize virtualization by partitioning the physical machine into multiple virtual servers.  Having many virtual servers running on a single physical server has many advantages.  If a virtual server becomes too large because of increased usage, administrators can easily move it to another physical server with more space.  The ease of virtual server replication allows virtual server snapshots to be used as file backups thus simplifying disaster recovery.</p> <p><img alt="Server VM Migration" src="/sites/default/files/files/vm/virtualserver.png" style="width: 507px;
