Loop Unrolling and Rescheduling

How Can Loop Unrolling and Rescheduling Work Together?

 

The goal of loop unrolling and rescheduling is to reduce the clock cycles per instruction (CPI) and therefore increase efficiency. The following examples build in complexity to help demonstrate how loop unrolling and rescheduling can reduce the CPI.

First, a few assumptions and a software-level loop to start the examples.

Assumptions:

(1) A 5-stage pipe-like-pipe is used during execution

(2) Forwarding is implemented

(3) Ignore control hazards

Software-Level Loop:

for (int i = 0; i < 1000; i++) {

a[ i ] = b[ i ] + c[ i ];

}

(Without Unrolling and Without Rescheduling)

The only stall in the above assembly code happens at instruction 3. The value at the memory address in 0(R3) will not be forwarded until clock cycle (CC) 5. There are 13 CC and 8 instructions for a CPI of 1.625.

There are two stalls in the MIPS code above. The first stall occurs when instruction 3 must wait for instruction 2 to forward the value from R3’s memory address. The second stall occurs when instruction 7 must wait for instruction 6 to forward the value from (R3 + 4 bytes)’s memory address. In Example Two there are 19 CC and 12 instructions for a CPI of 1.583. Loop unrolling alone has improved the CPI from 1.625 to 1.583. In addition to the lower CPI, loop unrolling has also decreased the number of branch predictions from 1000 to 500. The reduced branch predictions is a significant improvement that is not depicted in the 1.583 CPI.

 

 

Example Three

(With Rescheduling and With Loop Unrolling)

 

Loop unrolling decreased the CPI of the MIPS code by computing two additions per each iteration of the loop. Since the instructions are nested in a way that create read-after-write (RAW) hazards, the MIPS code stalled during two cycles. Rescheduling independent instructions to avoid stalls while maintaining the correct results can decrease CPI. The MIPS code below reschedules Example Two to avoid the RAW hazards:

All the RAW hazards in this loop have been removed by rescheduling the instructions after unrolling the loop. In Example Three there are 16 CC and 12 instructions for a CPI of 1.333.

Conclusion

The purpose of loop unrolling and rescheduling is to obtain a CPI as close to 1 as possible. As seen above, both techniques can be used together to achieve this goal. First, unrolling the loop once resulted in a deceased CPI by reducing the overhead cost of each iteration. Second, rescheduling removed the hazards in the unrolled loop. After both techniques were applied to the loop the CPI was reduced to 1.33 and reduced the total branch predictions by half.
