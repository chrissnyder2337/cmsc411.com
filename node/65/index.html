Introduction:

     Cloud storage is growing in popularity because of the fast rate at which the need to store information is growing. With the popularity of social media sites like Facebook, Twitter, LinkedIn, and others combined with  web giants like Google growing, archive data is booming, and the demand for large capacity storage at a low cost ad at an all time high. Thus due to supply and demand, the cloud memory storage service is growing. This page gives an overview of cloud memory storage.

The topics covered are: what 'The Cloud' is, general architecture, how the cloud performs, availability and reliability , and security of the Cloud.

What prerequisite knowledge does a reader of this website need?
Knowledge of traditional computer storage, and basic computer networking.

What is the Cloud?

     Cloud storage is information storage as a service versus ownership. The Cloud doesn't sound like a traditional name for data storage, however it is an abstraction of traditional storage using an interface. Storage can be acquired on demand from the user, the actual location of the storage can be remote, or local. Cloud storage can support multiple users, and a diverse range of data. The Cloud is different then traditional disk or even flash storage because it uses applications that are executed by the users web browser to access and store information. With Cloud computing there is no need for an installed application on the users hardware. Service providers rent the applications so they have full control over them, eliminatting the need for annoying updates, installs, and memory usage by the user. Any computer with the power to access the web can use cloud storage, most commonly laptops, desktops, mobile phones, tablets, even t.v's. A more intuitive name for cloud computing is 'Storage as a Service' (SaaS), but the name that has stuck with it, and is most commonly used is 'The Cloud'.

 

Structure of the Cloud:

     The primary purpose of cloud storage is to deliver instant memory storage to a user on demand in a flexible way. The architecture of Cloud memory is layered The first layer is known as the front end, it supplies users with a way to access the cloud, by exporting the API, usually the SCSI protocol. Then the middle layer of the Cloud is an algorithim which handles where to place the information. The final layer is the physical servers and computers that hold the information.

     Cloud computing is abstraction of the traditional client server model. Instead of owning one's own data servers, a user leases whatever amount of space he/she needs from a provider. This is an improvement because with the cloud you get exactly the amount of space you need, and only have to pay for that. This is a big advantage for business's because they can pay for 5 machines during working hours, and drop down to 1 during off hours, as opposed to managing their own 5 servers 100% of the time. The key difference is that someone else is managing the storage, and willing to lease whatever you need on demand. In comparison the traditional client server model is like buying a property, while cloud is like renting, in some cases when your going to be using the reasource a lot, and it is important to protect its a good idea to buy (Like a home), in other cases when you're going to be sharing the reasource a lot, and not always using it, renting is the better option (like office space).

Cloud Performance:

     The main factor in cloud performance is the transfer of information from the service provider to the user. The work for processing and storing data can be delegated to external hardware usually belonging to the service provider. Therefore there is no need for the user to have the processing power locally to have good performance. The only thing a user needs to ensure fast service, is reliable and fast network access.

     Traditional WAN transfers transfer data with bandwith ranging from low to 1 Gbps, acting as a bottleneck for data transfer. Because of this transfering data in bulk using the cloud was slow until Cloud service providers implemented new ways of transfering data, specifically for the purpose of the Cloud. Most service providers like Google, and Microsoft use web based protocals which use an object based design added on to HTTP. For example Amazon, and Asperasoft created a faster more efficient way to tranfer data via the Cloud, namely FASP, or Fast and Secure Protocal. FASP utilizes UDP, as opposed to TCP, which allows the host to manage the congestion, moving it to the application process in FASP. FASP increases the performance to 104.2 Gbps according to Aspera.

 

Cloud Availability:

     Availability is a key topic for Cloud computing because, nothing can beat the availability of good old solid state local memory storage, it is 100% available excluding hardware malfunctions, the user has 24/7 access to their data. The cloud must find a way to compete with that around the clock availability. After a user uploads his/her data onto the cloud, it must be able to provide that data back to the user at any time or else cloud storage is unreliable, and impractical for many applications. The main problem facing cloud service providers with regards to availability is network outages.

     One solution to network outages is information dispersal. Information Dispersal, or (IDA) is an algorithm created by Michael Rabin, which splits data into pieces encoding it with Reed-Soloman correction codes so that it can be pieced back together given a subset of the original data. The only negative of IDA is the processing requires a lot of overhead and can be intensive without help from hardware.

      Network outages can be devistating, recently Inuit's site went down, causing its users to lose access to valuable data and services such as Quicken, and TurboTax. Another more conservitive approach to outages, is hybrid storage. Hybrid storage combines both cloud and local storage to provide users the comfort of knowing their data is stored on in the cloud as well as on their machine. Apple for example offers it's iTunes users the option to store their music in the cloud. Hybrid computing is convienient because it is not hardware dependant, if something were to happen to your phone, or laptop you wouldn't lose all of your precious data. 

 

Cloud Security:

      With the popularity of Cloud computing growing, so should concerns for its security. Security becomes a much more complex issue when using the cloud because its no longer an issue of simply protecting your disk, or flash memory devices. With the cloud your information is out in the world somewhere and susceptible to other kinds of threats, including attackers trying to steal information, rapidly changing cloud architectures, the consumerization of storage, and regulatory environments.

     As the use of SaaS grows, so does web based organized crime. No longer are attackers simply hacking away for personal fame, with such valuable information being stored in the cloud, there is a growing population of attackers who are motivated by financial gain. Cyber criminals usually target the weak in comparison registration system used by Cloud service providers who have limited fraud detection capabilities. Cloud providers compensate by using strict initial registration requirements, and follow up verification. These additional steps when accessing your data on the cloud can slow the process down, but for now are the best solution to keep your sensitive information safe.
