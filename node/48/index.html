Loop Unrolling

Loop Unrolling

Loop Unrolling also known as loop unwinding is a technique that attempts to improve the execution speed of a program at the cost of increasing the length of the code. The process of loop unrolling focuses on increasing the number of commands in the body of a loop while reducing the number of loop control statements.

 

For example in a loop in high level language like Java:

for(int i = 0; i < 12; i+=1){
extrenalFunction(i);
}

Can be unrolled as:

for(int i=0; i < 12; i+=3){
externalFunction(i);
externalFunction(i+1);
externalFunction(i+2);
}

As a result of unrolling the loop, the every second and third comparison of the loop control mechanisms are skipped without any undesired side effects.

When we examine a similar situation using a pipelined MIPS machine the problem becomes even more obvious when looking at a simple algorithm to calculate R1 factorial (assume R1 greater than 1 and even)

Loop: DMUL R2, R1, R2
DSUBI R1, R1, #1
BNEZ R1, Loop
 
When performed on a pipelined machine the first major disadvantage can be noticed that a control hazard is created. When a branch condition is executed it may or may not change the program counter to something other than the current instruction plus 4. In a traditional five stage pipeline a branch instruction can not be identified until the Instruction Decode phase, the second stage of the five stage pipeline. This creates an effect where the machine must stall the next Instruction Fetch until it is known where the next instruction is. As a result, one stall for every branch will yield a preformance loss of 10% to 30% depending upon the frequency of the branch instruction.
 
In order to deal with this hazard there are are a variety of techniques that can be used to elminate the stall. One such technique is to have the machine predict which branch will be taken. The inherent risk with this strategy is that it requires that the pipe be flushed if an incorrect guess is made. One method that can be used to reduce this penalty is to use loop unrolling, this allows the program to bypass this cost. Employing this technique will allow program to keep the pipeline full.
 
Loop: DMUL R2, R1, R2
DSUBI R1, R1 #1
DMUL R2, R1, R2
DSUBI R1, R1 #1
BEZ R1, Loop
