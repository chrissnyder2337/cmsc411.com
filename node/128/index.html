Physical Components

Since we are working with physical components we are truly dealing with a finite amount of resources. In order to make these finite resources work together, CPU engineers must develop efficient designs that must not only work correctly, but also efficiently so that they are competitive in the computing industry. Before we look into CPU logic optimizations, lets look at the components of the CPU.

Control Unit - This component of the CPU that fetches, decodes, manages execution, and stores the resulting data from the several CPU stages. The Control Unit does this through timing and control signals which notify and instruct other components to perform certain tasks.

The ALU (Arithmetic Logical Unit) - This component of the CPU performs integer arithmetic and logical operations. The ALU pulls data from input registers and if given the instruction from the Control Unit, the ALU loads data through several digital circuits that perform the chosen operation. CPU engineers may choose to include more sophisticated operations instead of the usual mathematical operations like add/sub/mult/div/xor. Square root functions can be added, for example, which can complete in one clock cycle similar to the aforementioned operations.

Registers - These high speed, low-latency, small-sized data storage entities are common across all CPUs. Nearest to the ALU and Control Unit, registers may contain integer or floating-point data for completing their respective operations.

Cache Memory - Is similar to registers in that they hold relatively small pieces of data, however cache memory is slower due to the overhead required in accessing, reading and writing the data. In modern processors, there might be up to 3 levels of cache, each of which has increasing spatial locality.

As mentioned before, these basic elements of the CPU are physical and as such are constrained by size and speed. Due to these constraints, engineers have implemented certain optimizations which increases CPU performance. One optimization technique that engineers implemented was parallelism. Data, thread, and instruction level parallelism allow for the CPU to process increased amounts of data.
