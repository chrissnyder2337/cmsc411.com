ARM Family of Processors

The following are the most common ARM processors that were developed over time.

 

ARM1: This processor was the first commercialized RISC processor and it contained 25,000 transistors; ARM1 was primarily used in the BBC micro and a PC machine. Moreover, in comparison to ARM2, ARM1 has neither the R8 and R9 banked in the IRQ mode nor a multiply instruction.

 

ARM2: This is the follow-up to ARM1. Essentially, the additions of MUL and MLA enabled the use of real-time digital signal processing. These changes helped a lot with generating sounds. Moreover, ARM2 was the fastest RISC processor when the first ARM-based machines were starting to sell. In fact, ARM2 was arguably the fastest processor around because of the RISC design. Nonetheless, ARM2-based machines faced several challenges because proponents of IBM's PC were not willing to make a change.  

ARM3: ARM3 improved on ARM2 by providing 4K of cache and the SWP instruction. ARM3 is actually not faster than ARM2. In fact, both processors average 0.56 MIPS/MHz. However, ARM3 has a speed boost thanks to higher clock speed and a larger cache.

 

ARM4 AND ARM5: These processors do not exist because the numbering system for the processors was altered during the change from Acorn to ARM Ltd. Nevertheless, proponents of ARM believed that the RISC-based architecture was the epitome of sleek with a powerful RISC design.

ARM610: The ARM610 brought two new features to the ARM3. Firstly, the processor allowed full 32-bit addressing. Secondly, the the processor ushered in a new era of ARM based hardware. Even though the ARM610 provides high-level RISC performance, its fully static design ensures lower power consumption; therefore, A610 was ideal for embedded systems.

ARM710:  ARM710 provided an increased cache size of 8K rather than 4K. It also increased the clock speed to 40 MHz and improved the write buffer. ARM710 also supported Thumb mode, which allows the code to be smaller, and potentially faster if a small amount of memory is available.

ARM8: The processor improved over ARM7 by taking the 3-stage pipeline and turning it into a 5-stage pipeline. The change has nearly doubled the performance, while keeping the efficiency at level. Since one could integrate ARM8 with other peripherals, it means that ARM8 has great design flexibility.

ARM9: Another ARM processor that is an improvement over ARM7. By spending transistors, ARM9 is able to make key improvements of ARM7. The main improvement is decreasing heat production and lowering overheating risk. This is very important because for an embedded system heat is a big issue.

ARM10: ARM10's goal was to double the performance of its predecessor, ARM8. Also, rather than supporting five-stage pipelining like in ARM9, ARM10 uses a six-stage pipeline. Moreover, ARM10 has the option of supporting the vector floating point unit, which increases floating-point performance tremendously; this ability actually adds a seventh stage to the pipeline. This means that more instructions can run simultaneously. Therefore, the overall execution time will be faster.

ARM11: ARM11 added multiprocessor support and a cache architecture to ARM10. It also significantly improves the pipelining from ARM9 and ARM10. It also added SIMD instructions, which double the speed of the MPEG-4 and audio digital signal processing algorithms.
