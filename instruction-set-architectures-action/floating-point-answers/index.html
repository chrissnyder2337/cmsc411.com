Floating Point Answers

Because having problems without answers is retarded and barely helps you understand.

1: Represent these mixed number in binary. Do not use a calculator and show your work. Goto 2^-12 place.

  1. 1710 =    17/2 = 8R1
                  8/2 = 4R0
                  4/2 = 2R0
                  2/2 = 1R0
                  1/2 = 0R1
                  100012

       12810 = 2^710
       17/128 = 100012 >>7 = 0.00100012

  1. 1010 = 10102
    12/10102 = (10/1010) / 10 = (100/1010) / 100 = (10000/1010) / 10000 = 0.0001 R110 / 1010 = 0.00011 R10 / 1010 = 0.000110 R100 / 1010 = 0.0001100 R1000 / 1010 = 0.00011001 R110 / 1010 (pattern here) = .000110011001

 

  1. 2110 = 101012 , 2010 = 101002
    10101/10100 = 1 R1/10100 =  1.0000 R10000 / 10100 = 1.00001 R1100 / 10100 = 1.000011 R100 / 10100 = 1.0000110 R1000 / 10100 = 1.00001100 R10000 / 10100 (pattern here) = 1.00001100110011

2: Answer these questions.

  1. There are 32 bits in a single precision 32-bit word.
  2. There are 23bits in the significand (32-8 exponent bits – 1 sign bit = 23)

3: Convert these single-precision floating-point formatted numbers into decimal numbers. Bias of 127. The | and whitespace are simply there to help make it easier to read.

  1. 0 sign bit = positive number
    Exponent 10000010 = 13010 – 127 = 310
    Mantissa1.1012 << 3 =11012 = 1310
    Thus 13.0
  2. 1 sign bit = negative number
    Exponent 11110102 = 12210 – 127 = -510
    Mantissa 1.100110011001100110011002 >> 5 = .0001100110011001100110011
    -0.000110011001100110011 = 2^-4 + 2^-5 + 2^-8 + 2^-9 + 2^-12 + 2^-13 + 2^-16 + 2^-17 =
    1/16 + 1/32 + 1/256 + 1/512 + 1/4096 + 1/8192 + 1/65536 + 1/131072 = 0.0999984741 = ~0.1 = 1/10
  3. 0 sign bit = positive number
    Exponent 100100102 = 14610 – 127 = 1910
    Mantissa 1.11101000011001101010102 << 19 = 11110100001100110101.0102 = 111101000011001101012 + .0102 = 2^19 + 2^18… + 2^-2 = 100024510 + 1/410
    1000245.25
